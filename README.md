# Система управления контрактами и финансами

## 📋 Описание

Полноценное веб-приложение для управления контрактами, финансами, задачами и документами ГУП "Московский метрополитен".

## ✨ Возможности

- ✅ **Аутентификация** - вход по email и паролю
- ✅ **Управление контрактами** - просмотр и редактирование контрактов
- ✅ **Финансовый учёт** - отслеживание стоимости работ и оборудования
- ✅ **Управление задачами** - создание и отслеживание задач
- ✅ **Документооборот** - управление письмами и документами
- ✅ **История изменений** - полный лог кто и когда изменил данные
- ✅ **Облачное хранилище** - данные сохраняются на облачном сервере
- ✅ **Резервные копии** - автоматические резервные копии БД
- ✅ **Совместный доступ** - готовность к добавлению команды

## 🏗️ Архитектура

### Backend
- **Node.js** + Express - REST API
- **PostgreSQL** - база данных
- **JWT** - аутентификация
- **bcryptjs** - хеширование паролей

### Frontend
- **React** - пользовательский интерфейс
- **Tailwind CSS** - стилизация
- **Lucide React** - иконки

## 🚀 Развёртывание

## 📊 Структура данных

### Таблицы базы данных
- **users** - пользователи системы
- **contracts** - контракты
- **objects** - объекты в контрактах
- **tasks** - задачи
- **letters** - письма и документы
- **audit_log** - лог всех изменений

## 🔐 Безопасность

- Пароли хешируются с помощью bcrypt
- Все запросы защищены JWT токенами
- HTTPS на продакшене (автоматически на Heroku и Vercel)
- CORS настроен для безопасности

## 🔄 История изменений

Все изменения логируются в таблице `audit_log`:
- Кто внёс изменение (user_id)
- Когда внёс (timestamp)
- Что изменил (action, table_name, details)
- Можно восстановить любое состояние данных

## 🤝 Добавление команды

Когда будете готовы добавить команду:

1. Добавьте на фронтенде форму регистрации
2. Создайте новых пользователей через `/api/auth/register`
3. Установите правила доступа для разных ролей
4. Начните отслеживать кто и что изменяет

## 📞 Поддержка

При возникновении проблем:
1. Проверьте логи: `heroku logs --tail`
2. Проверьте переменные окружения: `heroku config`
3. Проверьте, что база данных инициализирована
